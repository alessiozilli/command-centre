What Goes Where — Decision Guide

for Forge sessions + local AI
A Forge lesson, rule, or business decision
→ Write to cloud mem0 (stays forever, searchable)
AZCK Ltd/AI/Forge/Identity/Forge_Lessons.md + mem0
A passoff or session handoff
→ Write to Google Drive Ops
AZCK Ltd/AI/Ops/Passoffs/YYYY-MM-DD_HH-MM_*.md
A file to deploy to the live website
→ Stage in Outputs/, then Alessio pastes to GitHub
AZCK Ltd/AI/Ops/Outputs/ → GitHub edit URL
A brand asset (logo, sigil, palette)
→ Save to Assets/, log in MANIFEST.md first
AZCK Ltd/AI/Forge/Assets/
A task or plan for forge-code or forge-cowork
→ INSERT into Supabase forge_tasks or forge_plans
Supabase → agent_messages bus if cross-instance
Something private — personal notes, experiments
→ Feed manually to Private Brain (Alessio only, not Forge)
D:\AI\Brains\Private\ · port 8052 · never syncs
A backup of Forge's cloud memories
→ Sync script runs cloud→Local automatically
D:\AI\Brains\Sync\Public-to-Local.py
A critical key or credential
→ Offline only, never in Drive or cloud
D:\AI\Vault\ (beast only)
